الفوتة عالسيرفر

طيب، السيرفر صار عندك — كيف بتفوتي عليه من جهازك؟ في جملة سحرية اسمها SSH. بتفتحي التيرمنال عندك بجهازك وبتكتبي:

🎯 الهدف: تفتحي سطر أوامر جوا السيرفر نفسه وإنتِ قاعدة على جهازك — كأنك ركّبتي شاشة وكيبورد عالسيرفر اللي بألمانيا.
ssh root@65.108.11.22
# بدلي الرقم بالـ IP تبع سيرفرك — بيوصلك بالإيميل لما يصير عندك سيرفر
🕵️ شو بصير خلف الكواليس؟ ▾

SSH بيفتح قناة مشفرة بين جهازك والسيرفر — كل أمر بتكتبيه بسافر مشفر، بنتفذ هناك، والنتيجة بترجع لشاشتك.

أول مرة رح يسألك سؤال ثقة طويل عريض — اكتبي yes وخلصينا 🙏

بعدها الباسورد. وبس تكتبيه مش رح يبين ولا حرف عالشاشة — هيك تصميمه، مش خربان، اكتبي وتوكلي على الله.

شو يعني SSH من جوا وكيف شغال؟ ولا تحملي هم هلأ — محطوط بقسم «زيادة خير» تحت. المهم اليوم: انسخي، الصقي، افتحي.

أول Deploy بإيدك

قبل أي كوماند — شو فكرة الـ deploy أصلاً؟ الوصفة وحدة بأي مشروع بالعالم، ثلاث خطوات بس:

1️⃣ بتجيبي الكود على السيرفر (من GitHub).
2️⃣ بتنزلي متطلبات مشروعك — المكتبات اللي بعتمد عليها.
3️⃣ بتشغليه (وإذا مشروعك بده «بناء»، بتبنيه قبل التشغيل).

الخطوة ١ نفسها للكل. أما ٢ و٣ بتختلف حسب الـ stack تبعك — يعني أول سؤال بتسأليه لحالك: مشروعي مبني بشو؟

Node / Reactnpm install للمتطلبات، وnpm run build للبناء
Pythonpip install -r requirements.txt وبتشغلي بـ python app.py أو شبهه
PHPcomposer install
HTML ساكنولا إشي 🎉 بس بتنسخي الملفات وnginx بقدمهم مباشرة
⚠️ كل الأوامر الجاية بهاي الصفحة مثال لمشروع Node. إذا مشروعك من عالم ثاني، بدلي أوامر التنزيل والبناء بأوامر عالمك من الجدول — والباقي كله (git، deploy.sh، nginx...) بضل نفسه حرفياً.

وصلنا للحظة الحقيقة 🧿 إنتِ جوا السيرفر — يلا نطبق الوصفة. أول مرة بتعملي clone (نسخة كاملة)، وكل مرة بعدها pull (بس التحديثات):

🎯 الهدف: نجيب الكود من GitHub لجوا السيرفر، وننزل متطلباته ونبنيه — هاي نسخة الـ Node من الوصفة.
# أول مرة بس:
git clone git@github.com:username/mashro3ek.git
cd mashro3ek

# كل مرة في تحديث جديد:
git pull
npm install
npm run build
# وبعدين شغلي بالطريقة اللي مشروعك بدها
🕵️ شو بصير خلف الكواليس؟ ▾

git ما بنسخ المشروع كله كل مرة — الـ pull بجيب بس الفروقات من آخر مرة، عشان هيك أسرع بكثير من الـ clone.

بس استني — GitHub بده يتأكد مين إنتِ. عشان السيرفر يقدر يسحب الكود (خصوصاً إذا الريبو private)، بدنا نعرّف GitHub عليه. ٣ أوامر وخلص:

🎯 الهدف: نولّد «هوية» للسيرفر (مفتاح SSH)، وننسخها لحساب GitHub تبعك عشان يتعرف عليه ويثق فيه.
# جوا السيرفر — ولدي المفتاح (اضغطي Enter على كل الأسئلة):
ssh-keygen -t ed25519

# اعرضي المفتاح العام وانسخي كل السطر اللي بيطلع:
cat ~/.ssh/id_ed25519.pub

خدي النص اللي طلع (بيبلش بـ ssh-ed25519) وروحي على GitHub من متصفحك:

صورتك فوق يمين ← Settings ← SSH and GPG keys ← New SSH key ← الصقي المفتاح بخانة Key ← Add SSH key. خلص ✅

🎯 الهدف: نتأكد إنه GitHub صار يعرف سيرفرك قبل ما نكمل.
ssh -T git@github.com
# إذا رد عليكي بـ Hi واسمك — نجحنا
🕵️ شو بصير خلف الكواليس؟ ▾

المفتاح زوج: خاص (بضل بالسيرفر وما حدا بشوفه) وعام (اللي لصقتيه بـ GitHub). كل مرة السيرفر بحكي مع GitHub، بثبت إنه معه الخاص اللي بركب على العام — متل قفل ومفتاحه.

هلأ يا غالية، بيظبط كل مرة تفوتي تكتبي بإيدك git pull وnpm install والقافلة كلها ورا بعض. وبيظبط كمان تحطيهم كلهم بملف واحد اسمه deploy.sh وتكتبي سطر واحد بس:

🎯 الهدف: نجمع خطوات النشر كلها بملف واحد — النشر بصير كلمة وحدة بدل خمس أوامر بتنكتب بالغلط.
# deploy.sh — حطي جوا أوامر الـ stack تبعك، هاي نسخة Node:
#!/bin/bash
echo "يلا بلشنا"
git pull
npm install
npm run build
echo "خلصنا، روحي شوفي موقعك"
🕵️ شو بصير خلف الكواليس؟ ▾

السكربت مش سحر — ملف نصي عادي فيه نفس الأوامر، وbash بقراهم سطر سطر بالترتيب، كأنك كتبتيهم بإيدك بس أسرع.

مرة وحدة بس بتعطيه إذن التشغيل، وبعدها كل deploy صار سطر واحد:

🎯 الهدف: لينكس ما بشغّل أي ملف كبرنامج إلا بإذن صريح — منعطيه الإذن مرة وحدة وخلص.
chmod +x deploy.sh   # مرة وحدة بالعمر
./deploy.sh          # كل مرة بدك تنشري
🕵️ شو بصير خلف الكواليس؟ ▾

الـ +x يعني executable — صلاحية بتنحفظ مع الملف نفسه. اعملي ls -l وبتشوفيها كحرف x جنب اسمه.

تكتبي git pull وnpm install والقافلة بإيدك كل مرة
‎./deploy.sh وبتروحي تعملي شاي 🍵

البواب غانم (Firewall)

السيرفر تبعك مش وكالة بدون بواب. في زلمة غانم واقف عالباب اسمه firewall — شغلته يقرر مين بفوت ومين بضل برا 😤

الأبواب نفسها اسمها ports: الـ 80 والـ 443 للويب، والـ 22 للـ SSH تبعتك. مرات بتضطري تفتحي port (عشان الناس توصل لموقعك) أو تسكري port (عشان الفضوليين ما يوصلوا).

إذا مش متذكرة شو يعني port — هو الباب المرقّم اللي بفوت منه نوع معين من الزوار. خلص، هذا كل اللي بدك ياه هلأ.

لا تتعمقي بغانم هلأ — لمحة وبكفي. لما تخلصي، قسم «زيادة خير» تحت فيه الشرح الأعمق والفيديوهات.

من IP بشع لدومين محترم

هلأ موقعك شغال، بس الناس بتوصله هيك: http://65.108.11.22

أولاً شكله زي رقم حساب بنكي 🙃 ثانياً كل العالم شايفة عنوان سيرفرك عالمكشوف — وهذا risk. أكيد ممكن تقولي «no risk no fun» — إحنا سمينا الصفحة على اسمها أصلاً — بس هالوضع تحديداً ما بيمشي.

الحل قطعتين: دومين (الاسم الحلو) وDNS (دفتر تلفونات النت — اللي بحول الاسم الحلو للرقم البشع). بتشتري الدومين من أي registrar، وبتحكيله «هذا الاسم يودّي على هذا الـ IP». خلص، هاي كل قصة الـ DNS.

وهون بدخل عالقصة واحد محترم اسمه Cloudflare: بيتصرف كـ DNS تبعك، وكمان كـ proxy. شو proxy؟ البواب الواقف قدام الدار — الزوار كلهم بيحكوا معه هو، وما حدا بشوف عنوان دارك الحقيقي. وفوق هيك بعطيكي قفل الـ https ببلاش 🔒 بونس نظيف.

موزّع الزوار (Nginx)

جوا السيرفر نفسه في زلمة ثاني واقف عالباب اسمه Nginx — وهذا اللي بسموه reverse proxy. شغلته: الزائر بوصل، وهو بوزّع — «هذا بده الموقع؟ عالـ frontend. هذا بده /api؟ عالـ back». هيك بيقدر الاثنين يعيشوا على نفس السيرفر وكل واحد ياخد زواره.

(في كمان إشي اسمه forward proxy — هداك بواب بشتغل عندك إنتِ مش عند السيرفر. مش موضوعنا اليوم، بس هلأ صرتي بتعرفي إنه موجود 🤓)

بيحتاج راسك لـ Nginx؟ هلأ مش ضروري تتعلميه. اللي بدك تعرفيه إشي واحد: إله ملف configuration. كل ما بدك تعدلي إشي، بتفوتي على الملف، بتعدلي، بتفحصي، بتعملي reload. خلص:

🎯 الهدف: نعدل إعدادات nginx، نفحصها إنها سليمة، ونفعّلها — كله بدون ما الموقع يطفي ولا ثانية.
sudo nano /etc/nginx/sites-available/default   # عدلي
sudo nginx -t                                  # افحصي إنك ما خربتي إشي
sudo systemctl reload nginx                    # فعّلي التغيير
🕵️ شو بصير خلف الكواليس؟ ▾

reload بخلي nginx يقرا الإعدادات الجديدة وهو شغال، بدون ما يقطع عالزوار الموجودين — عكس restart اللي بطفي وبولع.

وشكل الملف من جوا؟ إشي هيك — انسخيه وبدلي الأرقام:

🎯 الهدف: نحكي لـ nginx مين يودي وين — الزائر العادي عالـ frontend، وطلبات /api عالـ back.
server {
  listen 80;
  server_name mashro3ek.com;

  location / {
    proxy_pass http://localhost:3000;   # الـ frontend
  }
  location /api {
    proxy_pass http://localhost:5000;   # الـ back
  }
}

وإذا حابة من الأيام تكتبي configuration من الصفر بفهم — تم، الله يقويكي، الرابط تحت. بس اصبري هلأ، هديها 🖐️

وDocker؟ ولا كلمة عن Docker اليوم 🙅‍♀️ إشي إله يومه.
One does not simply يكتب nginx config من الراس — الكل بنسخ. الكل. حتى صاحب اللينكد إن اللي حاط Solutions Architect 💼

SSH بجد — مفاتيح بدل باسوردات

بدل ما تكتبي باسورد كل مرة، مفتاح بيفتحلك لحاله. أأمن وأريح، وهو نفس المنطق تبع مفتاح GitHub اللي عملتيه.

غانم بالتفصيل — ufw والـ ports المفتوحة

كيف تحكي مع البواب بنفسك: تفتحي وتسكري ports بـ ufw، وتشوفي مين فاتح باب بدون علمك: ss -tulpn | grep LISTEN

خلي التطبيق يضل شغال (pm2)

هلأ إذا سكرتي التيرمنال، التطبيق بموت معه. pm2 بخليه شغال لحاله وبصحّيه إذا وقع — زي أم بتصحّي ولادها عالمدرسة.
